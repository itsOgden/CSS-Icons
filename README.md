# @itsogden/css-icons

A build tool that converts SVG files into a single CSS file using data URIs and CSS custom properties. Includes a Vue 3 / Nuxt 3 component for type-safe icon usage.

Icons render via CSS `mask-image`, which means they inherit `color` and work naturally with Tailwind utility classes like `text-red-500`.

---

## How it works

1. You point it at a folder of SVGs via a config file
2. Running the generator produces two files in your project:
   - `css-icon.css` — all icons as CSS custom properties + mask rules
   - `icons.ts` — a TypeScript type union of every icon name + a `getIconClass` helper
3. A Nuxt plugin registers `<CssIcon icon="..." />` globally using those generated types

The generated files are committed to your repo. No build step is needed on fresh clones.

---

## Installation

```bash
pnpm add github:itsOgden/CSS-Icons
```

Then run the interactive setup wizard:

```bash
pnpm css-icons init
```

The wizard will:
- Create `css-icon.config.mjs` in your project root
- Create `plugins/css-icons.ts` (the Nuxt plugin)
- Add the `css-icons` script to your `package.json`
- Patch your `dev` and `build` scripts to run the generator automatically
- Print the one line you need to add to `nuxt.config.ts` manually

## Manual setup (if you prefer)

If you'd rather not use the wizard, here's what it does under the hood.

### 1. Config file

Create `css-icon.config.mjs` in your project root:

```js
export default {
  iconFolder: './assets/icons',   // where your SVGs live
  defaultFolder: 'duotone',       // see "Icon naming" below
  outDir: './plugins/css-icons',  // where generated files are written
  // backgroundFolders: ['colored'],
}
```

### 2. Nuxt plugin

Create `plugins/css-icons.ts`:

```ts
import { defineNuxtPlugin } from '#app'
import { defineComponent, h, type PropType } from 'vue'
import BaseIcon from '@itsogden/css-icons/component'
// @ts-ignore — generated by `pnpm css-icons`
import { getIconClass, type CssIconName } from './css-icons/icons'

const CssIcon = defineComponent({
  name: 'CssIcon',
  props: {
    icon: {
      type: String as PropType<CssIconName>,
      required: true,
    },
    useWidth: {
      type: Boolean,
      default: false,
    },
  },
  setup(props) {
    return () => h(BaseIcon, {
      className: `${getIconClass(props.icon)}${props.useWidth ? ' scale-width' : ''}`,
    })
  },
})

export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.component('CssIcon', CssIcon)
})
```

### 3. nuxt.config.ts

```ts
export default defineNuxtConfig({
  css: ['~/plugins/css-icons/css-icon.css'],
})
```

### 4. package.json scripts

```json
"scripts": {
  "css-icons": "node ./node_modules/@itsogden/css-icons/index.mjs",
  "dev": "pnpm css-icons --watch & nuxt dev",
  "build": "pnpm css-icons && nuxt build"
}
```

> **Windows users:** The `&` operator doesn't work in cmd or PowerShell. Install `concurrently` and use this instead:
> ```bash
> pnpm add -D concurrently
> ```
> ```json
> "dev": "concurrently \"pnpm css-icons --watch\" \"nuxt dev\""
> ```
> The `build` script is fine as-is on Windows since `&&` (sequential) works in both environments.
>
> The setup wizard will ask about your OS and handle this automatically.

## Running the generator

```bash
# Generate once
pnpm css-icons

# Watch mode — regenerates whenever an SVG is added, changed, or removed
pnpm css-icons --watch
```

After setup, `pnpm dev` runs the watcher automatically alongside Nuxt.

## Usage

```html
<CssIcon icon="duotone-gear" />
```

The `icon` prop is fully typed — your editor will autocomplete and error on unknown icon names.

### `useWidth` prop

By default icons are `1em × 1em` (square). For icons with non-square viewBoxes, add `useWidth`:

```html
<CssIcon icon="duotone-bars" useWidth />
```

### Sizing and color

Icons inherit `font-size` and `color` from their parent, so Tailwind classes work directly:

```html
<CssIcon icon="duotone-gear" class="text-2xl text-blue-500" />
```

---

## Icon naming

Icon names are derived from their file path relative to `iconFolder`.

| File path | Icon name |
|---|---|
| `duotone/gear.svg` | `duotone-gear` |
| `duotone/arrow-right.svg` | `duotone-arrow-right` |
| `colored/logo.svg` | `colored-logo` |

### `defaultFolder`

If you set `defaultFolder: 'duotone'`, icons inside that folder drop the folder suffix from the icon name:

| File path | Without defaultFolder | With defaultFolder: 'duotone' |
|---|---|---|
| `duotone/gear.svg` | `duotone-gear-duotone` | `duotone-gear` |

### Full-color icons

SVGs that use their own colors (logos, illustrations) should go in a folder listed under `backgroundFolders`. These render via `background-image` instead of `mask-image`, preserving their original colors but losing the ability to tint via `color`.

```js
backgroundFolders: ['colored']
```
